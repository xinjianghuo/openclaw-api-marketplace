## 1. 执行摘要

ChurnBuster 是一个 OpenClaw 技能，自动检测 Stripe 失败支付并智能恢复，帮助电商/SaaS 企业减少 10% 收入流失。核心功能：

- 实时监控 Stripe 支付失败事件
- 自动重试策略（智能间隔降级）
- 客户邮件通知（模板化 + 个性化）
- 替代支付方式建议（PayPal、加密货币等）
- 仪表盘报告（恢复率、收入拯救）

**定价**：$49/月 或 $499/年  
**预期 MRR**：首年 $5-10k（100-200 客户）  
**启动成本**：$0（零成本，OpenClaw + Stripe API）  
**开发周期**：2 周 MVP

---

## 2. 问题验证

### 痛点强度

- **行业数据**：在线业务约 10% 支付失败率（Stripe 平均）
- **影响**：失败支付若不恢复，直接收入损失；客户流失风险
- **现有方案**：
  - 手动处理（客服团队）→ 耗时、易出错
  - Stripe Radar（内置）→ 基础，缺乏主动恢复
  - 第三方工具（ChurnBuster, RecoverPay）→ 定价 $49-99/月，功能复杂

### 支付意愿

- 小型电商：愿意付 $29-49/月节省 5 小时/周
- SaaS 企业：愿意付 $79-99/月恢复高价值订阅
- **访谈目标**：找到 15 个确认支付意愿 ≥$30/月

---

## 3. 解决方案

### MVP 功能集

1. **自动检测**：监听 Stripe webhook `payment_intent.payment_failed`
2. **重试引擎**：
   - 立即重试（1次）
   - 智能退避（1h, 6h, 24h, 3d）
   - 卡片更新提醒（邮件 + 链接）
3. **邮件模板**：
   - 友好通知 + 一键重试
   - 替代支付选项（PayPal、加密）
4. **仪表盘**：
   - 恢复率、拯救收入、失败趋势
5. **集成**：
   - OpenClaw 技能（自动运行）
   - Stripe CLI 测试模式

### 技术栈

- OpenClaw skill（Node.js ~200 行）
- Stripe API（webhooks + payment_intents）
- 邮件：SendGrid 免费层（100 天/100 邮件）
- 数据存储：SQLite（本地，无服务器成本）
