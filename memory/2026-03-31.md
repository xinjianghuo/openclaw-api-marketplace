## 🚨 08:14 - Vercel Production 超时事件

### 警报
用户收到 Vercel 日志：`/api/run` 在 300 秒后超时。

### 根因分析
`lib/skills/node-doctor.js` 使用 `child_process.exec` 执行：
- `openclaw gateway status`
- `ping -n 1 gateway.openclaw.ai`

**Vercel serverless 环境禁止子进程**，导致调用挂起直到超时。

### 紧急修复（08:14-08:20）

1. **KV 超时控制** (`lib/license-manager.js`)
   - 所有 Upstash KV fetch 调用增加 `AbortController`，5秒超时
   - 防止 KV REST API 无限等待

2. **移除子进程调用** (`lib/skills/node-doctor.js`)
   - 完全删除 `child_process.exec` 依赖
   - `checkGatewayStatus()`: Vercel 环境返回状态说明
   - `testConnectivity()`: 使用 `fetch` 到 `/api/health`，3秒超时
   - 纯异步实现，无阻塞

   - 删除未使用的 `verifyPurchase` (Gumroad) 函数
